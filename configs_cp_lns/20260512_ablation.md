# Ablation Study Design: Fm|prmu|ΣTj

## Research Context

Two novel algorithmic components are proposed for the permutation flow shop total tardiness problem:

- **LB-derived dispatching rules** (LBE, LBL, LBA): three job-priority orders extracted from the minimum-cost-flow solution of the preemptive last-stage relaxation.
- **PW-CP** (Prefix-Window Constraint Programming): an iterative sliding-window CP engine that interleaves CP subproblem solving with insertion improvement, replacing the earlier (ineffective) BD-CP.

The following established methods constitute the known-method backbone across configurations:

- **EDD** – Earliest Due Date priority rule
- **NEH-MS** – NEH insertion heuristic with makespan tie-breaker, adapted for total tardiness
- **II(1)** – Insertion Improvement (single-job, best-improvement, multi-pass until convergence)
- **Base-CP** – Final CP-SAT refinement of the full permutation model

### C1 — Bottom Line

`EDD → Base-CP`

Classical priority-rule dispatching followed by a pure CP-SAT solve. Serves as the practical performance floor: what a practitioner obtains by simply feeding an EDD sequence into a modern CP solver.

### C2 — 2010s SOTA Baseline

`EDD → NEH-MS → II(1, multi-pass) → Base-CP`

Best known heuristic pipeline from prior work: EDD-informed NEH construction, followed by first-improvement insertion local search, with final CP refinement. This configuration contains **no novel component** and represents the strongest baseline achievable with known methods.

### C3 — +LB-derived Dispatching Rules

`EDD + LB-Disp(LBE/LBL/LBA) → NEH-MS → II(1, multi-pass) → Base-CP`

Differs from C2 only in the initial dispatching phase: the best permutation among EDD and the three LB-derived orders (LBE, LBL, LBA) is selected before NEH-MS and subsequent improvement.

> **C2 → C3 isolates the contribution of the LB-derived dispatching rules.**

### C4 — +PW-CP

`EDD → NEH-MS → repeat [ PW-CP, II(1, 1-pass) ] → Base-CP`

Differs from C2 by adding the PW-CP loop after NEH-MS. Inside PW-CP, the CP subproblem optimizes a sliding window of `B=7` jobs (`profile_fixed_cnt=0`) with adaptive step size (`step_on_improve=7`, `step_on_no_improve=1`), interleaved with a single II pass (`max_passes=1`). The outer `repeat_while_improvement` loop ensures convergence.

> **C2 → C4 isolates the contribution of PW-CP.**

### C5 — IGC (Full Method)

`EDD + LB-Disp(LBE/LBL/LBA) → NEH-MS → repeat [ PW-CP, II(1, 1-pass) ] → Base-CP`

Both novel components combined. Serves as the full IGC algorithm.

> **C3 → C5** cross-validates the PW-CP effect under the LB-dispatching baseline.  
> **C4 → C5** cross-validates the LB-dispatching effect under the PW-CP baseline.

## Key Comparisons

| Comparison | Question |
|------------|----------|
| C1 → C2   | How much do NEH-MS + II improve over pure CP-SAT? (context only) |
| **C2 → C3** | **Does the LB-derived dispatching improve solution quality over EDD?** |
| **C2 → C4** | **Does PW-CP improve solution quality over known heuristics alone?** |
| C3 → C5   | Does PW-CP remain effective when combined with LB-dispatching? |
| C4 → C5   | Does LB-dispatching add value on top of PW-CP? |

## Parameter Summary

| Component | Key Parameters |
|-----------|---------------|
| EDD | sort by `d_j` ascending |
| LB-dispatch | MCF solve on preemptive last-stage relaxation → extract LBE (earliest), LBL (latest), LBA (average) orders → dispatch each, retain best |
| NEH-MS | NEH insertion with `tie_breaker = makespan` |
| II (multi-pass) | `subseq_size=1`, `first_improvement=false`, `max_passes` unset (convergence) |
| PW-CP | `added_batch_size=7`, `profile_fixed_cnt=0`, `step_size_on_improve=7`, `step_size_on_no_improve=1`, `max_time_per_add=10s`, `solver_thread_cnt=1` |
| II (1-pass) | `subseq_size=1`, `first_improvement=false`, `max_passes=1` |
| Base-CP | `computational_time=null` (remaining budget), `solver_thread_cnt=1` |
| All | `seed=42`, total wall-clock limit per instance controlled by `stopping_criteria.yaml` |
